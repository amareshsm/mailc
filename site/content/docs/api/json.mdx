---
title: JSON conversion
description: Convert between `.mc` markup and the JSON AST, and validate / normalize JSON templates programmatically.
---

mailc has a JSON representation of every email template — the same shape that drives the visual builder and the `compileFromJSON()` API. These helpers convert between markup and JSON, validate the JSON shape, and normalize older or hand-written documents.

```ts
import {
  jsonToMarkup,
  markupToJSON,
  validate,
  validateDocument,
  normalizeJSON,
} from 'mailc'
```

For the compile-from-JSON entry point itself see [`compileFromJSON()`](/docs/api/compile#compilefromjson). For the universal validator that accepts markup, JSON IR, or AST, see [`validate()`](/docs/api/validate).

## markupToJSON()

```ts
function markupToJSON(source: string): MCNode
```

Parses a `.mc` source string and returns the equivalent JSON AST. Useful for migrating existing markup into a visual builder, generating fixtures, or persisting templates as JSON.

```ts
import { markupToJSON } from 'mailc'

const node = markupToJSON(`
  <mc>
    <mc-body>
      <mc-section><mc-column>
        <mc-text>Hi {{user.firstName}}</mc-text>
      </mc-column></mc-section>
    </mc-body>
  </mc>
`)

// node = { type: 'mc-body', attributes: {}, children: [...] }
```

The returned `MCNode` is the same shape `compileFromJSON()` accepts.

## jsonToMarkup()

```ts
function jsonToMarkup(node: MCNode, indent?: number): string
```

The inverse of `markupToJSON()` — serializes an `MCNode` back to readable `.mc` markup. When the root node is `mc-body` or `mc-head` it is wrapped in `<mc>...</mc>` automatically so the output is valid standalone markup.

```ts
import { jsonToMarkup } from 'mailc'

const markup = jsonToMarkup(node)
```

`indent` is the starting indentation level (defaults to `0`); pass a non-zero value when embedding the result inside another document.

## validate(node)

```ts
function validate(input: string | ASTNode | MCNode): ValidationResult
```

Validating a JSON IR tree is one of three forms accepted by the universal [`validate()`](/docs/api/validate). Pass an `MCNode` and the function dispatches to the JSON walker (structural rules — required parents, allowed children, attribute types, duplicate IDs, and so on). Returns the same `MCIssue` shape used by `compile()`.

```ts
import { validate } from 'mailc'

const result = validate(node)
if (!result.isValid) {
  for (const err of result.errors) console.error(err.code, err.message)
}
```

## validateDocument()

```ts
function validateDocument(doc: MCDocument): {
  isValid: boolean
  errors: MCIssue[]
  warnings: MCIssue[]
}
```

Accepts a full `MCDocument` (the wrapper that adds `version` + `metadata` around the template tree) and runs document-level checks (required metadata fields, `dataSchema` vs `sampleData` cross-validation) in addition to the inner tree validation. Use this when the JSON came from a saved file rather than an in-memory builder.

## normalizeJSON()

```ts
function normalizeJSON(input: MCNode): MCNode
function normalizeJSON(input: MCDocument): MCDocument
```

Returns a new tree (or document) with legacy / hand-written JSON normalized into the canonical shape — fills in default `attributes`/`children` arrays, coerces missing fields, and is safe to call before `validate()` / `compileFromJSON()`. The original input is not mutated.

```ts
import { normalizeJSON, compileFromJSON } from 'mailc'

const safe = normalizeJSON(rawJsonFromUser)
const result = compileFromJSON(JSON.stringify(safe))
```

## When to use what

| Goal | Use |
|---|---|
| Take a `.mc` file and store it as JSON | `markupToJSON()` |
| Render JSON back to readable markup | `jsonToMarkup()` |
| Check a tree before compiling | [`validate(node)`](/docs/api/validate) or `validateDocument()` (full doc) |
| Accept third-party / older JSON safely | `normalizeJSON()` then [`validate()`](/docs/api/validate) |
| Compile a JSON template to HTML | [`compileFromJSON()`](/docs/api/compile#compilefromjson) |
